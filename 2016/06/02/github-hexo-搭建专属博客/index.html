环境搭建

安装node.js https://nodejs.org/en/
安装github windowhttps://desktop.github.com/
建议安装github window不用单独安装git,github window安装后，可以直接用git shell直接操作，后续很方便
安装hexohttps://hexo.io/zh-cn/官网里面有详细的安装提示。
申请github账号，略过。

购买域名

购买域名地址，可以支持支付宝https://sg.godaddy.com/zh/。先搜索你想要的域名，一般是搜索自己的姓名，如图所示：

搜索出来后，加个合适的直接用支付宝支付购买。

域名解析管理

用的是dnspodhttps://www.dnspod.cn/,注册账户后，添加域名，并且添加记录，如图所示

NS2条记录是dnspod默认的记录，会自动导入。

  • 添加 主机记录为@ 类型为A的记录 记录值这个Ip为对应github **.github.io的地址。获取方式如下图所示
  • 添加www记录 类型为CNAME 记录值对应github respotory的名称 列如：zhansan.github.io

域名设置

在godday里面设置一下域名服务器：登录goddaddy账户,进入域名管理，在域名服务器里面设置一下。具体操作如图所示：,这二个值对应DNSPOD里面二条系统记录的值。

CNAME文件

  1. 在本机hexo 文件夹下得source子目录下，创建CNAME文件，写入自己的域名 列如：tongruyi.club。

问题

  1. ERROR Deployer not found: github
    修改_config.yml后，hexo deploy 如果报这个错误。
    解决方法：
    npm install hexo-deployer-git –save
  2. 域名解析失败注意点：
    DSNPOD有没有添加对应的域名以及记录
    godday要设置域名服务器
    hexo/source/CNAME该文件有没有创建
    hexo/.config.ymlw文件deploy字段属性分别设置为如下图所示
  3. 本地图片加载——方便高效的方式
    https://github.com/CodeFalling/hexo-asset-image 首先确认 _config.yml 中有 post_asset_folder:true。
    在hexo 目录下，执行
    npm install https://github.com/CodeFalling/hexo-asset-image –save
    在\hexo\source_posts下面新建一个文件夹，文件夹名称与文章标题保持一致。将使用的图片全部放到该文件夹下。引用的时候采用的格式如下！[](文章题目/*.jpg)。

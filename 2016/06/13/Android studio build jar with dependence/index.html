Android studio build jar with dependence

Posted on 2016-06-13   |     |   visitors

我们项目组专门做能力开发的，前面的版本都是用eclipse开发的，后面迁移到android studio还是用eclipse里面的FatJar打包。这个问题拖了一个月，最近突击了一下，解决了这个问题。希望能够让更多人吸取经验。
对gradle不熟悉的可以看看 Gradle User Guide中文版https://dongchuan.gitbooks.io/gradle-user-guide-/content/tutorials/
英语能力强地也可以看看英文版 https://docs.gradle.org/current/userguide/userguide_single.html

一. 分析dependencies

指令gradle App:dependencies 可以列出所有的依赖树，通过依赖树可以分析出自己的Jar包需要包括哪些第三方jar。

二. 打包src下面的java文件

首先先来看看怎样将自己src文件夹下面的代码打包，不废话，直接贴出代码：

task buildJar(dependsOn: [build], type: Jar) {
appendix = "demo"
baseName = "androidJar"
version = "1.0.0"
classifier = "release"
//后缀名
extension = "jar"
//最终的 Jar 包名，如果没设置，默认为 [baseName]-[appendix]-[version]-[classifier].[extension]
archiveName = "littlec.jar"

//需打包的资源所在的路径集
def srcClassDir = [project.buildDir.absolutePath + "/intermediates/classes/release/"];
//初始化资源路径集
from srcClassDir

//去除路径集下部分的资源
//exclude "org/chaos/demo/jar/MainActivity.class"
//exclude "org/chaos/demo/jar/MainActivity/$*.class"
//去掉不需要打包的目录和文件
exclude('test/', 'BuildConfig.class', 'R.class')
//去掉R$开头的文件
exclude { it.name.startsWith('R$'); }

//去掉不需要打包的目录和文件
exclude('test/', 'BuildConfig.class', 'R.class', 'androidTest/')
exclude('META-INF/')
//去掉R$开头的文件
exclude { it.name.startsWith('R$');
//it.name.startsWith('org.apache.thrift');
it.name.startsWith('META-INF');
}
//注: exclude include 支持可变长参数
 exclude "im/"
}

这个task 里面可以将src里面的代码打包成指定的名称如littlec.jar，生成目录在/build/libs/下面。
注意点：
打包之前需要确认/INTERMEDIATES/CLASSES/RELEASE/下面存在文件，如果不存在则需要重新编译一下，重新编译BUILD->MAKE PROJECT

三. 将依赖远程仓库和本地依赖打包

Android工程里面有很多在线依赖以及项目工程里面libs下面的本地依赖，如果想将这些依赖和自己的代码打成一个Jar包，我们应该怎么办？这个点让我费解了好久，其实我们可以发现：依赖远程仓库的第三方包也会缓存到本地，那么我们怎么用gradle找到这些第三方依赖？后面查资料发现configurations.compile.collect就可以遍历依赖里面的jar包，我们只需要再看一下每个元素是不是还有子树，而zipTree可以遍历一个文件下面的子文件，所以上面Task 的from 可以改为

configurations.compile.
 collect {
it.isDirectory() ? it : zipTree(it)
}

输入明确了，只需要将每个依赖里面的META_INF目录去掉就行了。那就稍微改进一下：

 configurations.compile.
 collect {
it.isDirectory() ? it : zipTree(it).matching {
exclude{it.path.contains('META-INF')}
}
}

如果是文件夹对其文件路径都匹配一下META-INF，进而排除 META-INF文件夹。
上面排除了META-INF等不需要的文件，有时候我们不想将所有在dependencies里面依赖的第三方包打包，那么我们怎么很优雅地将这些包进行删选了？而且不影响正常的编译。只需要在代码里面加入

configurations{
retain
compile.extendsFrom retain
}

并且将dependencies里面将自己不想打包的compile 修改为 retain。如下所示：

dependencies {

retain 'javax.annotation:javax.annotation-api:1.2'

}

最后将上面from里面的代码修改如下：

`(configurations.compile-configurations.retain).
       collect {
        it.isDirectory() ? it : zipTree(it).matching {
            exclude{it.path.contains('META-INF')}
        }
    }`

这样javax.annotation:javax.annotation-api:1.2 这个包就不会被打进最终的Jar包。
最终的task代码如下，这个task 基于上述的buildJar，这样就可以将src下面的代码和第三方依赖的代码打成一个jar包。

task createJarWithDen(type: Jar) {
baseName = "littlec"
from {
 List list= (configurations.compile-configurations.retain).
   collect {
it.isDirectory() ? it : zipTree(it).matching {
exclude{it.path.contains('META-INF')}
}
}
   List<ObjectremoveList=new ArrayList<>();
for(Object object:list){
println(object.toString());
if(object.toString().contains("jsr")) {
removeList.add(object)
println("jsrObject remove")
}else if(object.toString().contains("javax.annotation")){
removeList.add(object)
println("annotation remove")
}else if(object.toString().contains("MiPush_SDK_Client")){
removeList.add(object)
println("MiPush_SDK_Client remove")
}
}
list.removeAll(removeList)
return list;
}
with buildJar
}

里面for循环是我们自己项目里面一些不用的项目，你们可以根据自己的需求更改。

四. 运行

gradlew createJarWithDen 或者 gradle createJarWithDen

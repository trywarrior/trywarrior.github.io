git常用操作

Posted on 2016-12-27   |     |   visitors

git虽然命令相当多，但是记住下面这些命令就够用了。这些都是使用频率很高的命令：

git init 初始git库

git add 添加文件到缓存区

git commit -m 提交到仓库

git status 查看当前的状态

git log 可以查看提交的记录

git reset -hard id 版本会退到某一个commit_id
commitid可以从git log查看

git diff HEAD – filename可以比较工作区和暂存区有啥的版本有啥区别

git checkout – filename或者git reset HEAD file 撤销修改操作

git rm filename 从版本库里面删除某个文件

git checkout -b dev 创建新分支

git branch 查看当前分支

git checkout -b dev新建并且切换分支到dev

git checkout branchname 切换分支

git branch -d 删除分支

git branch 创建分支

git merge 合并分支

git stash 暂时保存现场，这样工作区就是干净的，master也是 干净的

git stash apply 恢复现场

git stash pop 删除现场

git push remote-name

git checkout -b test origin/dev 将远程仓库dev分支拉取到本地,并且新建test分支。建立本地分支和git branch –set-upstream
branch-name origin/branch-name建立本地分支与远程的关联

git tag 打一个新标签

git tag 查看所有标签

git show 查看标签

git push original –tags 一次性推送全部尚未推送到远程的本地标签

git tag -d 删除本地标签

git push origin :refs/tags/ 可以删除一个远程标签
